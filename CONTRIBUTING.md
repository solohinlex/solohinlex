## 2. Как предложить изменения

### Сообщить об ошибке (Issue)

- Сначала проверьте вкладку **Issues** — возможно, проблема уже   зафиксирована.
- Если нет — создайте новый Issue: опишите ошибку, шаги для воспроизведения и что ожидалось в результате.
- **Нашли уязвимость безопасности?** Не создавайте публичный Issue — см. [Политику безопасности](SECURITY.md).

### Pull Request

1. Сделайте **Fork** репозитория.
2. Создайте отдельную ветку: `git checkout -b feature/my-new-script` или `git checkout -b fix/typo-in-text`
3. Внесите изменения, напишите понятное сообщение к коммиту: `fix: исправить синтаксис в скрипте генерации`
4. Запушьте ветку в свой форк: `git push origin feature/my-new-script`
5. Откройте **Pull Request** в ветку `main`.

## 3. Требования к материалам

- **Скрипты** — код должен быть читаемым, сложные участки желательно прокомментировать. Весь код распространяется под лицензией **MIT**.
- **Тексты** — предложения должны соответствовать стилю проекта. Обратите внимание: на тексты распространяются отдельные условия из файла [LICENSE](LICENSE.md).
